什么是React Hook

Hook 是 React 16.8 的新增特性。它可以让你在不编写 class 的情况下使用 state 以及其他的 React 特性。
换而言之就是Hook能够是function加入react的相关特性。

npx是什么

Effect Hook

Effect Hook 可以让你在函数组件中执行渲染后的副作用操作。
useEffect Hook 也可以看作React Class生命周期中componentDidMount、componentDidUpdate、componentWillUnmount这三个函数的组合。

不需要清除的effect副作用

下面使用useEffect实现标题实时显示点击次数的功能

Effect Hook使得我们在编写代码时，不用考虑时在组件挂载、更新后是否执行，而是每次都去执行。每次都执行可能会造成资源的浪费，下面我会提到如何去控制它。
如果使用React Class组件，还得在componentDidMount、componentDidUpdate中间都绑定相同的函数，代码重复。

控制 Effect Hook(什么时候运行副作用)

上面鼠标跟踪器的例子可以知道，每次执行Effect会造成先删掉上次的Effect再执行这次Effect，但前后两次Effect之间并没有发生改变，造成不必要的资源的浪费。所以当我们需要的值在两次Effect之间没有变化时，我们就跳过这次Effect的调用。而React也提供控制Effect的API。

其实useEffect函数可以提供两个arg，而上述使用时我们只传入了一个回调函数，其实还有第二个参数，默认是该组件的state列表，只要组件的state发生改变，就回调用一次Effect。故实现上述鼠标跟随器不进行多次订阅的功能只需在useEffect函数添加一个空列表即可。

如果想要实现手动控制Effect，只需要添加一个state，通过改变state激活对应的Effect即可。

HOC-Higher order component 高阶组件 与 Hook 的异同

HOC并不是React的API，而是一种模式。高级组件就是一个函数，接受一个组件作为参数，返回一个新的组件。

$Component + HOC = Component with additonal fuctionality or data$
